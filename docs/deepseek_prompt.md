# DeepSeek 论文摘要总结 - 提示词配置

## 默认提示词

系统默认使用以下提示词对论文摘要进行一句话总结：

```
请用一句简洁的中文总结以下论文摘要的核心贡献和方法：

论文标题：{title}

摘要内容：
{abstract}

要求：
1. 只用一句话（不超过100字）
2. 突出论文的核心创新点或方法
3. 使用学术但易懂的语言
4. 直接给出总结，不要有任何前缀
```

## 自定义提示词

你可以在 `.env` 文件中设置 `DEEPSEEK_PROMPT` 来自定义提示词。

**注意**：提示词中必须包含 `{title}` 和 `{abstract}` 占位符，系统会自动替换为实际的论文标题和摘要。

### 示例1：更简洁的总结

```
DEEPSEEK_PROMPT=用一句话（50字以内）总结这篇论文的核心方法：{title}\n\n{abstract}
```

### 示例2：突出应用场景

```
DEEPSEEK_PROMPT=请总结这篇论文的核心方法和应用场景（100字以内）：\n\n标题：{title}\n\n摘要：{abstract}
```

### 示例3：技术细节导向

```
DEEPSEEK_PROMPT=请用技术语言描述这篇论文提出的关键技术创新点（80字以内）：\n\n{title}\n\n{abstract}
```

## 调整建议

1. **控制长度**：明确指定字数限制，避免AI生成过长的总结
2. **明确重点**：告诉AI你关注的是方法、创新点还是应用场景
3. **格式要求**：要求"直接给出总结"可以避免AI添加"这篇论文..."等前缀
4. **语言风格**：可以指定"学术"、"通俗"、"技术"等语言风格

## API 配置

1. 访问 [DeepSeek Platform](https://platform.deepseek.com/) 注册账号
2. 创建 API Key
3. 在 `.env` 文件中设置 `DEEPSEEK_API_KEY=your_api_key`

**费用说明**：DeepSeek API 按 token 计费，一篇论文摘要总结大约消耗 500-1000 tokens，费用很低。
